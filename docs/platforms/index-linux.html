Linux/XKB Platform Info

Most of the BigBag is implemented for Linux in the Big Bag for XKB repository on GitHub.

Briefly, I've hacked up a bunch of modifications to the X.Org xkeyboard-config files found in the xkb-data package, that define keyboard layout related settings for the X server:

  • Colemak[eD] and several locale/script variants as layout definitions; Unified Symbols (US) and Keep Local Symbols (KS) flavors of these
  • The Tarmak transitional layouts and Mirrored Colemak, likewise defined as layouts (found under the US locale)
  • Extend, implemented as lvl5–8 layout mappings together with a lvl5 modifier key that can be added to the CapsLock key using an option setting
  • Ergonomic mods mostly implemented as keyboard "models", so they may be applied to any layout – not just Colemak!
  • Some "soft" ergo mods implemented as xkb options instead; this is the case for the Curl-DH and Sym mods
  • There's an install BASH script that writes my modifications to the X11 xkb directory so you may choose them in settings or by commands
  • There's a setkb.sh script that can generate a setxkbmap command to activate one of the BigBag layout combos

You can check out the Big Bag (XKB) forum topic for more detailed info. It's a bit messy and outdated here and there, but Linux users are often clever and adventurous so I hope you manage.

If you just want to skip to the download (at your own risk), here is the GitHub Big Bag for XKB repository. It's got a little documentation and you can run its scripts with a -h tag for help but the main info is still here and in that Forum topic.


Notes:

  • The setkb script can help you activate a BigBag layout. It is not made for activating other layout setups, such as your current default.
  • To see what your current setup is, you can for instance run the command 'setxkbmap -v 9' (or whatever verbosity level suits you).
  • To get Colemak-DH right, you have to both use the 'misc:cmk_curl_dh' option itself, and an Angle keyboard model if you're on a rowstag keyboard.
